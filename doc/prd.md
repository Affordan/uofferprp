一、PRD：强基计划信息系统 Demo
1. 文档信息

项目名称
强基信息系统开发（Demo）

产品定位
一个面向考生、家长与升学研究者的“强基计划官方政策与分数线查询平台”，提供政策归档、官方来源追溯、分数线查询、学校对比、数据导出与后台维护能力。

版本范围
V1 Demo

目标形态
Web 站点 + 后台管理 + 数据库 + 自动化采集/导入链路 + AI 驱动研发与纠错体系

2. 项目背景

强基计划自 2020 年起实施，国家层面的根政策已经明确：面向服务国家重大战略需求的人才选拔培养，重点覆盖数学、物理、化学、生物以及历史、哲学、古文字学等方向，录取模式采用“高考成绩 + 高校考核 + 综合素质评价”，且高考成绩占比不得低于 85%。当前试点高校为 39 所。

但现实中的信息分散且不统一：

国家层面有教育部根政策与阳光高考专题；

省级层面常见的是教育考试院专题页、报考说明、公示，不一定每省都有统一的“强基总政策”；

高校层面有招生简章、入围标准、录取分数线、录取结果通知，但公开格式差异很大，有的给表格，有的给图片，有的只给查询入口。吉林大学公开“入围标准”，山东大学既有“入围分数线”又有“分省分专业录取分数线（综合成绩）”，浙江省教育考试院还会汇总 39 所高校简章。

因此，这个项目的本质不是普通资讯站，而是一个“数据产品 + 证据库 + 查询系统”。

3. 产品目标
3.1 业务目标

建立一个可展示国家、省份、高校三层强基计划信息的网站。

支持按省份、学校、年份、专业、分数线类型查询数据。

支持多高校同条件对比。

支持将查询结果导出为 PNG 和 Excel。

建立可持续维护的结构化数据库与后台管理能力。

3.2 产品目标

所有展示数据尽量带官方来源。

所有分数数据必须区分数据类型。

所有对比必须在相同条件下进行，避免误导。

页面风格现代、清爽、丝滑、可信赖。

3.3 非目标

V1 不做高并发推荐系统。

V1 不做复杂用户社区。

V1 不做完全无人值守的全自动采集。

V1 不做大而全的 AI 问答主入口。

用户当前遇到的问题：

7. 数据定义与标准

这个项目成败的关键在于“字段定义统一”。

7.1 分数线类型标准

必须将数据拆成以下类型：

入围线
依据高考成绩进入校测名单的最低标准。

校测结果
校考成绩或校测相关结果说明。

拟录取综合成绩
高考 + 校测 + 综合素质合成后的分数或最低线。

最终录取结果
正式录取名单或录取最低分信息。

结果通知类
只有结果公告、但未公开完整分数表的数据。

这样设计，是因为当前高校公开口径并不统一。官方页面里既有“入围标准”，也有“分省分专业录取分数线（综合成绩）”，还有仅提供平台查询的通知。

7.2 来源等级标准

A：教育部 / 阳光高考 / 省级教育考试机构 / 高校官方招生网

B：高校二级单位官方页面

C：权威媒体转载

D：非官方整理页

V1 前台默认只展示 A、B 级数据；C、D 仅供后台参考。

7.3 数据完整度

完整：分省 + 分专业 + 分值齐全

部分完整：有省份/专业，但字段缺失

非结构化：只有图片或公告

查询型：需登录平台查询
